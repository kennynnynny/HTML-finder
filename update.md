# Промт для Qwen: Добавление панели с информацией об элементе

```markdown
# Задача: Обновление HTML Copy on Hover — панель информации об элементе

## 📋 Контекст проекта
Работаю с браузерным расширением **HTML Copy on Hover** (Manifest V3, vanilla JS). 
Файлы: `manifest.json`, `content.js`, `popup/`, `assets/`.
Расширение подсвечивает элемент при наведении и копирует его HTML по клику с модификатором.

## 🎯 Цель обновления (v1.2)
При наведении на элемент (когда показывается blue overlay) **вверху страницы должна отображаться компактная панель** с информацией о выделенном элементе:

### Отображаемые данные:
| Поле | Пример | Источник |
|------|--------|----------|
| **Тег** | `div`, `button`, `img` | `element.tagName.toLowerCase()` |
| **Размеры** | `320 × 240` | `getBoundingClientRect()` → width/height (округлить до целых) |
| **ID** | `#main-content` или `—` | `element.id` (показывать с `#`, если не пустой) |
| **Class** | `.btn .primary` или `—` | `element.classList` (через точку, через пробел) |

### Пример отображения:
```
[ div ]  320 × 240  |  #header  |  .nav .primary
```

## ⚙️ Технические требования

### 1. Визуал панели
- **Позиция**: `position: fixed; top: 12px; left: 50%; transform: translateX(-50%)` — по центру вверху
- **Стиль**: 
  - Фон: `#fff9e6` (светло-жёлтый, в теме проекта)
  - Текст: `#333`, шрифт `monospace` или `system-ui`, размер `12-13px`
  - Отступы: `6px 12px`, border-radius: `6px`
  - Тень: `0 2px 8px rgba(0,0,0,0.15)`
  - `pointer-events: none` (не блокирует клики)
  - `z-index: 2147483647` (выше всего)
- **Анимация**: плавное появление/исчезновение (`opacity + transform`, 150ms)

### 2. Логика (в content.js)
- Создать элемент панели один раз при инициализации (как overlay)
- При `mouseenter`:
  - Обновить текст панели данными элемента
  - Показать панель (если скрыта)
- При `mouseleave`:
  - Скрыть панель
- При смене настроек через popup — не влияет на панель (она всегда активна)
- **Важно**: панель не должна влиять на работу highlight overlay и копирование

### 3. Код-конвенции
- Добавить в тот же IIFE в `content.js`
- Использовать существующий паттерн: создание элементов через `document.createElement`, стили через `Object.assign` или CSS-классы
- Все строки интерфейса — **на русском** (как в проекте)
- Добавить комментарии к новой логике
- Не ломать существующую функциональность

### 4. Пример кода для текста панели:
```javascript
function getElementInfo(el) {
  const tag = el.tagName.toLowerCase();
  const rect = el.getBoundingClientRect();
  const size = `${Math.round(rect.width)} × ${Math.round(rect.height)}`;
  const id = el.id ? `#${el.id}` : '—';
  const classes = el.classList.length ? `.${[...el.classList].join(' .')}` : '—';
  return `[ ${tag} ]  ${size}  |  ${id}  |  ${classes}`;
}
```

### 💡 Советы по использованию промта:
1. Скопируйте весь блок выше и отправьте в чат с Qwen.
2. Если Qwen предложит код — проверьте, что он вставлен в правильный контекст `content.js` (внутри IIFE).
3. После обновления протестируйте:
   - Наведите на элемент с `id` и `class` → панель должна показать данные
   - Наведите на элемент без `id`/`class` → должно быть `—`
   - Клик с модификатором должен по-прежнему копировать HTML
4. Если панель мешает — проверьте `z-index` и `pointer-events`.
